# 🏝️ 코지 아일랜드 — 개발 플랜 (실행용)

> 출처: `src/projects/cozy-island/docs/개발플랜.md`의 "앞으로 할 수 있는 작업" 후보를
> 실제 실행 가능한 단위로 쪼개 우선순위·완료조건과 함께 정리한다.
> 우선순위는 사용자 요청으로 언제든 변경 가능. 별도 지정이 없어 **광산 본구현**을 1순위로 둔다
> (이미 `data/mineLevels.json`이 존재하고, 로직만 스텁이며, 핵심 스태미나 루프에 직접 연결되기 때문).

---

## 현재 상태 요약 (시작점)

- 코어 루프(채집→스태미나 0→취침→최대치+1) 동작. 농사/동물/요리/건물/퀘스트/도감 구현됨.
- 월드: 단일 대륙 + 울타리 우리 + 돌길/게이트, 격자 A\* 길찾기, 정사각 픽셀 도트 렌더.
- 저장 `SAVE_VERSION = 7`.
- **스텁/미구현**: 던전(데이터/입구만), 패시브 UI 미노출, 밸런스 미검증.
- 데이터 자산: `data/mineLevels.json`, `data/dungeonLevels.json`, `data/passives.json` 존재.

## 마일스톤

### M1 — 광산 본구현 (1순위)
**목표**: 스텁 상태의 광산을 층별 채광 + 하강 루프로 완성.

- [x] `data/mineLevels.json` 스키마 확정/보강: 층(depth)별 등장 광석 가중치, 채광 스태미나 비용, 하강 조건.
- [x] `systems/MineSystem.ts` 신설: 현재 층 상태, 광석 노드 절차 생성(rng 시드 = 층+일자), 채광 처리, 하강/상승.
- [x] `GameState`에 광산 진행 상태(현재 층, 최고 도달 층) 추가 + 저장 연동 → 저장 구조 변경 시 `SAVE_VERSION = 7`로 상향.
- [x] 렌더: 광산 내부를 별도 씬으로 그리거나 우리 1칸을 광산 입구로 사용 (정사각 도트 유지).
- [x] 하단 메뉴 "광산" 버튼 → 입장/퇴장 UX, 층 표시 HUD.
- [x] 희귀 광석 → 인벤토리/상점 판매·레시피 연동.
- **완료조건**: 입장→채광(스태미나 소모)→하강→귀환→판매까지 한 사이클 플레이 가능, 저장/복원 정상.

### M2 — 던전 플레이 루프
**목표**: 데이터/입구만 있는 던전을 실제 플레이 루프로.

- [ ] `data/dungeonLevels.json` 검토, 클리어/보상 규칙 정의.
- [ ] `systems/DungeonSystem.ts`: 입장·진행·보상 처리 (전투 없이 채집/탐험형으로 시작).
- [ ] 입구 상호작용 + 진행 HUD, 보상 → 인벤토리.
- [ ] 저장 연동 (M1에서 올린 버전에 필드 추가, 필요 시 마이그레이션).
- **완료조건**: 입구 진입→진행→보상 획득→귀환 1사이클 동작.

### M3 — 콘텐츠 확장 ("우리 한 칸 추가" 규칙 준수)
- [ ] 신규 작물 1종: `data/crops.json` + 전용 우리(`worldLayout.json`/`layout.ts`) + 게이트.
- [ ] 신규 동물 1종: `data/animals.json` + 우리.
- [ ] 신규 레시피 1~2종: `data/recipes.json`, 신규 재료 연동.
- [ ] 퀘스트 라인 연장: 신규 콘텐츠를 안내하는 퀘스트 3~5개.
- **완료조건**: 추가 우리가 길찾기/게이트/카메라 스냅에 자연스럽게 편입, 퀘스트로 도달 가능.

### M4 — 밸런스 패스
- [ ] 첫 5분 루프 체감 측정 (채집 횟수/골드 획득/취침까지 시간).
- [ ] `data/economy.json`, `crops.json`(성장시간), `stamina.json`(비용) 조정.
- [ ] 초반 막힘 구간 제거 (첫 밭/첫 동물까지의 골드 곡선).
- **완료조건**: 신규 시작 → 첫 취침 → 첫 밭 구매까지 막힘 없이 진행.

### M5 — 시스템/UX 폴리시
- [ ] 패시브(`passives.json`) UI 노출 (도감 또는 별도 패널).
- [ ] 건물 업그레이드 시각/효과 다양화 (`upgradeRules.json`, `buildings.json`).
- [ ] 상황별 BGM 레이어 + SFX 보강 (`sound.json`, 절차적 오디오).
- [x] 1차 홈 구역 아트/HUD 패스: 핵심 스프라이트 실루엣 확대, 잔디/해안/펜 바닥 밀도 개선, 하단 메뉴 과밀 해소, 목표 퀘스트 HUD 추가.
- [ ] 모바일 UX: 패널 레이아웃 정리, 목표 안내 화살표/마커.

## 작업 규칙

1. 현재 로컬 브랜치는 `main`. 코드 변경을 시작하기 전 작업 브랜치를 만들거나 사용자가 지정한 브랜치로 전환한다.
2. 데이터 우선: 밸런스/콘텐츠는 `data/*.json`에 두고 로직은 데이터를 읽도록.
3. 구조 변경으로 구 세이브가 깨지면 `SAVE_VERSION` 상향 + (가능하면) 마이그레이션.
4. 렌더는 정사각 픽셀 도트 유지 (곡선/그라데이션 금지).
5. 각 마일스톤 완료 시 `PROGRESS.md` 갱신 + `docs/개발플랜.md` 완료 섹션에 반영.
6. `npm run build` + lint 통과 확인 후 커밋.
7. PR은 명시 요청 시에만 생성.
