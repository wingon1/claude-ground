# Tiny Dew Valley — 수정 방향 기획서

## 0. 한 줄 요약

**Tiny Dew Valley의 도트·픽셀 비주얼 정체성은 그대로 유지하되, 현재의 게임 로직(도구·시간 마감·신단·NPC 호감도)을 전부 걷어내고, 그 자리에 Cozy Island의 "탭 이동 + 근접 자동작업 + 스태미나 취침 성장" 루프를 이식한다.**

즉 결과물은 *"Cozy Island의 게임을 Tiny Dew Valley의 옷으로 입힌 것"* 이다.
화면에 보이는 색·패널·폰트·스프라이트 톤은 TDV 그대로, 손에 잡히는 플레이는 Cozy Island.

---

## 1. 유지할 것 — 디자인 / 도트 분위기 (그대로)

TDV의 시각 정체성은 손대지 않는다. `styles.css`의 토큰과 컴포넌트 스킨을 최대한 재사용한다.

| 항목 | 유지 내용 |
| --- | --- |
| 팔레트 | `--panel:#f6e9c9`, `--panel-edge:#b88a52`, `--panel-dark:#e3cfa0`, `--ink:#4a3422`, `--accent:#7cae4e`, 배경 `#1c2a1c` |
| 폰트 | `ui-monospace` 계열 (Cozy Island의 `system-ui`로 바꾸지 않음) |
| 캔버스 | `image-rendering: pixelated`, 절차적으로 그린 도트 스프라이트 톤 |
| 패널/모달 | `.tdv-panel`, `.tdv-modal`, `.tdv-card`, `.tdv-btn` — 둥근 모서리 + `box-shadow: 0 2px 0` 입체 버튼 느낌 |
| 토스트 | `.tdv-toast`(good/bad) 팝 애니메이션 |
| 타이틀/엔딩 카드 | `.tdv-title`, `.tdv-titlecard` 스킨 재사용(문구만 교체) |
| HUD 칩 | `.tdv-panel` 기반 골드/스태미나 표시 박스 |

> 핵심 원칙: **새 기능이 들어와도 UI는 항상 TDV의 양피지 스킨으로 렌더한다.** Cozy Island의 인라인 스타일(`#fff6e6` 등)을 그대로 가져오지 말고, TDV CSS 클래스/토큰으로 다시 입힌다.

## 2. 제거할 것 — TDV의 게임적 기능 (전부 삭제)

아래는 "게임적 기능"이므로 전부 제거한다. (해당 데이터/엔진 로직/該당 UI 포함)

- **도구 시스템 전체**: 괭이·물뿌리개·도끼·낫·손 (`ToolId`, `data/tools.ts`), 핫바 1~5번 도구 선택, 도구별 스태미나 코스트.
- **경작 파이프라인**: 땅 갈기(till)→씨앗 심기→물주기→수확의 수동 단계. `Tile.terrain('tilled')`, `wateredToday/Yesterday`, `daysUnwatered`, `hasFertilizer`.
- **물 자원**: 물뿌리개 물탱크(`water`/`waterMax`), 물 보충, 물뿌리개 업그레이드(copper).
- **작물 품질**: `CropQuality(normal/silver/gold/perfect)`, 품질 글리프(◇◆★), 품질별 판매가.
- **시간/마감 시스템**: 하루 분(`timeMinutes`), 시간대 표시(`period`), 06:00 기준 시계, **Day 28 마감**.
- **고대 신단(Shrine)**: `ShrineState`, 공물 번들(`data/shrineBundles.ts`), 복원 목표, **굿/비터스위트 엔딩 분기**.
- **NPC 친밀도**: 마을 주민 2명, 호감도/하트(`NPCFriendship`), 선물 등급(loved~hated), 대화/시간대 대사/마일스톤, HUD 하트 표시.
- **출하함 정산**: 밤 정산(`pendingShip`, `tally` 페이즈), 출하 보너스 언락.
- **기진맥진/기절**: `exhausted`, `nextDayStaminaCap`(쓰러지면 다음날 상한 감소).
- **기존 상점/제작 트리**: TDV의 씨앗 할인·배낭·작업대/공방(`workbench/workshop`) 언락 체계.

> 즉 `types.ts`의 `GameState`는 사실상 폐기 수준으로 재작성된다.

## 3. 가져올 것 — Cozy Island 게임 기능 (이식)

Cozy Island의 게임 루프를 TDV로 옮긴다. 데이터(JSON)·시스템 로직은 Cozy Island 모델을 기준으로 한다.

### 3.1 핵심 루프
- **탭 이동**: 화면을 탭하면 그 지점으로 캐릭터가 걸어간다. (TDV의 조이스틱/방향키 대신, 또는 병행 — 4.1 참조)
- **근접 자동작업**: 나무·바위·작물·동물 곁에 서면 도구 선택 없이 **자동으로 일을 시작**한다. (TDV의 "도구 들고 클릭" 폐기)
- **스태미나 → 취침 성장**: 행동마다 스태미나 소모. 0이 되면 텐트에서 잠. **깨어나면 최대 스태미나 +1**. **시간대·요일·마감 없음** — 진행은 오직 "자고 일어나기"로만 누적.

### 3.2 시스템별 이식 목록 (Cozy Island → TDV)

| 시스템 | 가져오는 내용 | 출처 |
| --- | --- | --- |
| 자원 채집 | 나무/바위/덤불/광석 노드, 내구도·리스폰, 자동 채집 | `data/resources.json`, `interactionRules.json` |
| 농사 | **작물별 밭 칸을 구매**해 늘리고 실시간(초) 성장·자동 수확/재심기 | `crops.json`, `farmPlots.json` |
| 동물 | 시설(닭장 등) 짓고 동물 구매 → 시간마다 산물 생산 → 수집 | `animals.json` |
| 요리 | 레시피 큐(조리 슬롯·craftSeconds), 발견 시스템 | `recipes.json` |
| 건설 | 건물 짓기 + 레벨 업그레이드, 언락 조건 | `buildings.json` |
| 상점 | 밭/동물 구매, 카테고리별 일괄 판매(자원/작물/축산물/음식/광물) | `shops.json`, `economy.json` |
| 퀘스트 | 목표(채집/수확/판매/건설/취침 등) + 골드·**젬** 보상 | `quests.json` |
| 광산 | 지하 층, 광석 노드, 더 깊이 하강(조건부) | `mineLevels.json`, `MineSystem` |
| 도감/수집 | 발견 아이템·레시피, 통계 기록 | `seenItems`, `counters` |
| 화폐 | 골드 + **젬(gems)** 2종 | `economy.json` |
| 패시브/언락 | 진행 카운터 기반 잠금 해제(`UnlockCondition`) | `passives.json`, `upgradeRules.json` |
| 사운드 | 절차적 BGM/SFX (선택) | `sound.json` — TDV의 `engine/audio.ts`로 재구현 가능 |

### 3.3 상태 모델
`GameState`를 Cozy Island 형태로 교체한다(핵심 필드):
`gold, gems, stamina, maxStamina, inventory[], plots[], animals[], buildings{}, cookQueue[], counters{}, itemTotals, seenItems, recipesDiscovered, quests{}, mineCurrentFloor/DeepestFloor, gameTime`.
→ TDV의 `day/timeMinutes/water/shrine/npcs/toolUpgrades/ending` 등은 제거.

## 4. 핵심 결정 사항 (충돌 지점)

### 4.1 조작 방식 — 탭 이동 vs 조이스틱
Cozy Island의 "근접 자동작업" 루프는 **탭 이동**을 전제로 설계돼 있다(서서 자동으로 일함). TDV의 조이스틱+핫바는 도구 시스템을 제거하면 의미가 약해진다.
- **권장**: 게임성 = Cozy Island이므로 **탭 이동 + 자동작업**을 채택. 모바일에서 조이스틱은 보조로 남길 수 있으나, 핫바(도구 선택)는 제거.
- HUD 우측의 메뉴/제작/가방 아이콘 버튼(`.tdv-iconbtn`)은 유지하되, Cozy Island의 하단 메뉴바(가방/상점/건설/광산/요리/메뉴)를 **TDV 스킨으로** 추가.

### 4.2 컨텍스트 액션 바
Cozy Island는 대상 근처에서 "베기/캐기/수확/재우기" 같은 컨텍스트 버튼을 하단에 띄운다. → TDV의 `.tdv-btn`/`.tdv-action` 스킨으로 재현.

### 4.3 진행 종료 조건
TDV는 "Day 28 + 신단"이라는 명확한 엔딩이 있었으나, Cozy Island는 **무한 성장형(엔드리스)**이다. → 엔딩 분기 제거, 타이틀의 "엔딩 보기"류 문구 정리. 타이틀 카드 자체는 유지하되 "포근한 섬 가꾸기" 톤으로 카피 교체.

## 6. 데이터 / 코드 구조 변경

```
src/projects/tiny-dew-valley/
  data/        ← 전면 교체: crops/animals/buildings/recipes/quests/mine/resources/economy 등
               (Cozy Island의 데이터 모델을 이식, 밸런스 수치는 그대로 시작 가능)
  engine/      ← game.ts: 게임 루프를 탭이동+자동작업+스태미나취침으로 재작성
               world.ts: 타일 경작 대신 노드/밭/펜 배치
               sprites.ts: 도트 톤 유지하며 필요한 스프라이트 추가
               save.ts: 새 GameState 스키마로 saveVersion 올림
               audio.ts: 유지(선택적으로 절차 BGM)
  ui/Overlay.tsx ← HUD/하단탭바/컨텍스트버튼/바텀시트 패널을 TDV 스킨으로 재구성
  styles.css   ← 토큰/스킨 유지, 하단 탭바·컨텍스트바 클래스 추가
  types.ts     ← GameState를 Cozy Island 모델로 교체
```

- `src/projects/registry.ts`의 TDV 카드 `description`을 새 플레이에 맞게 갱신(마감/신단/NPC 문구 삭제, "탭 이동·자동작업·스태미나 취침 성장" 강조). `tags`는 유지.
- 저장 호환: 기존 세이브는 스키마가 완전히 달라지므로 **마이그레이션하지 않고 버전 불일치 시 새로 시작**한다(`saveVersion` 상향).

## 7. 작업 단계 (구현 시 순서 제안)

1. **스펙 확정**: 본 문서 + 조작 방식(4.1) 결정.
2. **타입/상태 교체**: `types.ts`의 `GameState`를 새 모델로. 저장 버전 상향.
3. **데이터 이식**: `data/` 하위 JSON/TS를 새 시스템에 맞게 채움(작물/동물/건물/레시피/퀘스트/광산/자원/경제).
4. **엔진 재작성**: 탭 이동·근접 자동작업·스태미나 취침·실시간 성장·광산 모드.
5. **UI 재구성**: HUD·하단 탭바·컨텍스트 버튼·바텀시트 패널을 TDV 스킨으로.
6. **제거 정리**: 도구/물/신단/NPC/시간마감/엔딩 관련 코드·데이터·UI 삭제.
7. **레지스트리/카피 갱신** 및 동작 점검.

## 8. 비범위 (Out of Scope)

- 다른 프로젝트(예: cozy-island 원본) 수정 — 본 작업은 TDV 폴더 안에서만.
- 멀티플레이, 신규 아트 에셋 외주, 시간대/날씨 시스템 신규 추가.
- 기존 TDV 세이브 데이터 마이그레이션.
